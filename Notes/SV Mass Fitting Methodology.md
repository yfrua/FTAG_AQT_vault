# Prerequisite knowledge 
Fitting formula:
![](assets/SV%20Mass%20Fitting%20Methodology/file-20260226182700.png)
- i stands for i-th NN discriminant interval.
- $N_{i, MC}$ is the predicted flavour-inclusive event yield for each discriminant interval; $C$ is a global normalization factor and $F^f$ are the jet-flavour fractions; $P^f_i (m_{SV})$ is the probability density function of $m_{SV}$ taken from simulation.
- The $C$, $F^f$ and $SF^{b,light}_i$ parameters are allowed to float in the fit, while $N_{i, MC}$ and $\epsilon^f_{i, MC}$ are fixed to the predictions from simulated events and $SF^c_i$ is set to $1.0 \pm 0.3$.¬Ý
- There are $5\times 3=15$ data points (100-85%, 85-77%, 77-70%, 70-65%, 65-0% for WPs and no SV, 0-3 GeV, 3-6 GeV for SV mass) with 2 constraint of $\sum_i \epsilon_{i, MC}^f \times SF_i^f = 1, f=l, b$ (data efficiency adds up to unity) and one one $\sum_f F^f=1$, and $1+3+6*2=16$ parameters to fit. And everything above goes down to a DoF of $15-16+3=2$.

- This formula is different from $SF=\epsilon_{data}/\epsilon_{MC}$, can we deduce it from the fitting formula?
	- No, they are two different formulas, fitting formula is trying to fit MC into data from 3 different level, 0-D scalar $C$, 1-D vector $F^f$ and 2-D tensor $SF_i^f$, while the last one can be interpreted as scale factor.

# Analysis code breakdown 
- codes in [light-jet_calib_dijet](https://gitlab.cern.ch/fye/light-jet-calib-dijet)
- following summary is generated by AI:

## 1. Plot.cxx ‚Äî Diagnostic Plotting

Purpose: Produces diagnostic plots from histogram data for validation and inspection.

Main flow:
- Reads a config file, sets up Atlas style, creates `HistReader` to load histograms
- Loops over pT bins (inclusive + 9 bins), taggers, and optionally tag-bin splits
- Calls three plot functions:

`PlotSample` ‚Äî Stacked histograms comparing Data vs MC (Dijet, Zjets, Wjets, ttbar), split by flavor (b/c/light). Creates data/MC ratio plots. Scales MC to data integral if `NormaliseToData` is set. Handles both 1D and 2D (tag-bin split) histograms with projections.

`PlotFlip` ‚Äî Compares "nominal" vs "flipped" tagger discriminant distributions (e.g., Jet_GN3PflowMuonsV00 vs Jet_GN3PflowMuonsV00Flip). Normalizes both to unity. Draws working point cut lines (70%-90%). Used to validate the flip method.

`PlotFlavour` ‚Äî Overlays normalized distributions for b, c, and light-flavour jets separately, computing and printing flavour fractions with statistical errors.

Also draws B-tag working point thresholds (70%-90%) on relevant plots.

## 2. ProcessInputs.cxx ‚Äî Template Extraction

Purpose: Extracts and normalizes 1D template histograms from ntuple-derived root files, writing them to an output file for workspace building.

Main flow:
- Reads config, creates `HistReader`
- Loops over: jet collections ‚Üí taggers ‚Üí pT bins ‚Üí tag bins
- For each combination, calls `GetTemplate()` which:
	- Projects a 2D histogram (`TagBin_vs_Jet_VxMass`) into a 1D template per tag bin
	- Rebins to 3 bins (mSV: -3 to 0, 0 to 3, 3 to 6 GeV)
	- Forces negative bin contents to zero (from Sherpa negative weights)
	- Produces both raw and unit-normalized histograms
- Simultaneously extracts `Sherpa` (`mc_Sh`) and `MadGraph` (`mc_MG`) templates for b, c, l flavors, plus data and inclusive MC templates for closure tests
- If tracking systematics are enabled (`TemplateSys != "doNominal"`), applies tracking uncertainty variations via `MakeTrackingSysTemplate()` ‚Äî reads ratio files and multiplies nominal templates by variation/nominal ratios
- Writes all templates to `ContNegTagInputs.*.root`

## 3. BuildWS.cxx ‚Äî Workspace Construction

Purpose: Builds a `RooFit`/`HistFactory` workspace encoding the negative-tag control region fit model.

Main flow:
- Reads template file from `ProcessInputs`
- For each (generator, collection, tagger, pT bin, systematic variation) combination:
`BuildWorkspace` creates a `HistFactory::Measurement` with:
- Channels: One per tag bin (TagBin1‚ÄìTagBin6/7)
- Preprocess functions: Computes expected yields via equations like` N_Inc_Pretag * Scale * f_b * Eff * SF_Neg` for each flavor
	- TagBin1 (tagged): `N_Inc * Scale * f * (1 - Sum(Eff_i * SF_Neg_i))` ‚Äî the negative tag region
	- Other bins:` N_Inc * Scale * f * Eff_i * SF_Neg_i`
- Parameters: `N_Inc_Pretag` (normalization), `Eff_MC_TagBin*_flavor` (MC efficiencies), `SF_Neg_TagBin*_flavor` (scale factors to float), `f_b`, `f_c` (flavor fractions), `Scale`
- Data: Observed data histograms (or MC for closure test mode)
- Templates: Normalized MC templates per flavor per channel, with stat error activation
- Efficiency computation: Calculates `Eff = N_pass / N_pretag` from MC histograms, with error propagation
- MCMC SF application: Optionally applies MCMC-based scale factors from calibration files to adjust MC efficiencies
- Saves efficiency histograms and 2D efficiency maps

`ComputeFlavorFractions` ‚Äî Saves pre-fit flavor fractions (`f_b`, `f_c`) and yields per pT bin and pT-tagbin combination.

## 4. DoFit.cxx ‚Äî Performing the Fit

Purpose: Fits the workspace model to data, extracting scale factors for all flavors across all pT bins and tag bins.
Main flow:
Process ‚Äî For each workspace:
- Retrieves fitted parameters (SFs, efficiencies, f_b, f_c, Scale)
- Constructs FixedCut SFs via RooFormulaVar: SF_FixedCut_i = Sum(SF_Neg_j * Eff_j for j>=i) / Sum(Eff_j for j>=i), converting from tag-bin definition to fixed-efficiency working point (90%, 85%, 80%, 75%, 70%)
- Constructs TagBin1 SF as (1 - Sum(SF_j * Eff_j)) / Eff_1
Fit ‚Äî For each fit option (NOM, MC_STAT, DATA_STAT, CSF_UP/DOWN, BSF_UP/DOWN):
- Resets nuisance parameters to nominal snapshot
- Fixes/selective free based on fit option:
- MC_STAT: all gamma (stat) nuisance params fixed
- DATA_STAT: only light-flavour SFs free
- CSF_UP/DOWN: charm SFs shifted ¬±30%
- BSF_UP/DOWN: b SFs shifted ¬±30% (if fixBSF)
- For non-nominal systematics, all SFs fixed except those matching the systematic
- Runs pdf->fitTo() with retry logic (strategy 2, Minuit) if status ‚âÝ 0
- Plots post-fit distributions (data, stacked components, fit uncertainty band, ratio plot)
- Correlation plots
- Saves SF values, efficiencies, f_b, f_c, and Scale into 2D result histograms (pT √ó tagbin) with errors from both RooFitResult::error() and propagated errors for derived quantities
## 5. DoResults.cxx ‚Äî Uncertainty Decomposition & Output
Purpose: Assembles final scale factors, decomposes uncertainties, and produces output files and plots for the CDI (Calibration Decision Interface).
Main flow:
ProcessUncertainties ‚Äî For each observable (SF_b, SF_l, f_b, f_c):
6. Reads result histograms from all fit options (NOM, MC_STAT, DATA_STAT, CSF¬±, BSF¬±, alternative generator, template systematics)
7. Reads MC efficiency uncertainties from MistagEffMC file
8. Reads double-ratio correction histograms for flip‚Üídirect extrapolation (from tracking systematics comparison)
9. Decomposes total uncertainty by subtracting in quadrature:
- Syst_Gen: alternative generator difference (Sherpa vs MadGraph)
- Syst_Charm: symmetrized CSF UP/DOWN variation
- Syst_BSF: symmetrized BSF UP/DOWN variation
- Syst_EffMC: MC efficiency relative error √ó SF value
- Syst_TRK_*: tracking template systematics (TRK_RES_D0Z0_MEAS, s4415-4417)
- Stat_Data: from DATA_STAT fit
- Stat_MC: MC stat inferred from NOM¬≤ - MC_STAT¬≤
- Syst_Total: total systematic
5. Delta method: For SF extrapolation beyond fitted range (TagBin6‚ÜíTagBin7 for light, TagBin3‚ÜíTagBin4 for c in Continuous2D), uses SF_new = SF_old * alpha where alpha is a constrained nuisance parameter. Uncertainties are propagated from the reference bin.
6. PlotScaleFactor ‚Äî Plots SF vs pT for each working point
7. PlotUncertainties ‚Äî Plots uncertainty breakdown (stat, syst, per-source)
8. PlotFlavour ‚Äî Plots flavour fractions f_b, f_c vs pT
9. MakeInputFileCont ‚Äî Generates CDI input file with signed systematic variations (preserving sign for unitarity)
Key outputs:
- Result root file with decomposed uncertainty histograms
- Plots of SFs, uncertainties, flavour fractions
- CDI-compatible input text files with all systematic variations and signs
